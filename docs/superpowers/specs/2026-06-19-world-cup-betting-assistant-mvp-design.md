# 世界杯竞彩决策助手（MVP）设计规格

## 1. 产品定位

世界杯竞彩决策助手（MVP）是一个本地网页 App，帮助完全不懂足球的用户在 30 秒内完成以下判断：

- 查看今日世界杯比赛
- 获得推荐投注方向
- 获得推荐串关方案
- 获得风险提示
- 决定今天是否下注

产品定位是“竞彩辅助决策工具”，不是预测神器。第一版不承诺命中，不提供自动下注、自动出票或支付能力。

## 2. 已确认方案

第一版采用“本地网页 App + 示例世界杯数据 + 真实 API 预留口”的方案。

- 应用形态：本地网页 App
- 技术路线：可演示 SPA，后续可替换真实数据源
- 数据来源：第一版使用结构化示例数据，预留 API-Football / Football Data API 接入位置
- 推荐金额：显示固定小额“示例投入”
- 首页风格：30 秒决策仪表盘

选择该方案是为了最快验证核心体验：用户是否能在打开页面后快速理解“今天看什么、哪些谨慎、是否值得小额参与”。

## 3. 范围

### 3.1 第一版要做

- 今日比赛列表
- “开始分析”按钮
- 今日稳胆 TOP3
- 今日价值投注 TOP3
- 今日避坑比赛
- 推荐串关方案
- 比赛详情页或详情弹窗
- 评分拆解
- AI 风格解释文案
- 风险提示与免责声明
- 空状态和数据缺失状态
- 桌面与手机宽度适配

### 3.2 第一版不做

- 用户系统
- 登录
- 支付
- 聊天
- 社区
- 历史数据库
- 自动下注
- 自动出票
- 自动投注
- 用户预算管理

## 4. 用户流程

1. 用户打开本地网页 App。
2. 首页自动显示今日世界杯赛事。
3. 页面展示比赛双方、开赛时间、当前盘口或盘口占位。
4. 用户点击“开始分析”。
5. 系统基于示例数据和规则评分生成今日推荐。
6. 用户查看稳胆、价值投注、避坑比赛和串关方案。
7. 用户点击任意比赛或推荐项进入详情。
8. 用户阅读球队数据、评分拆解、推荐理由和风险提示。
9. 用户自行决定今天是否下注。

## 5. 首页设计

首页采用“打开即决策”的仪表盘布局。

顶部区域：

- 今日世界杯赛事数量
- 今日比赛列表
- 开赛时间
- 比赛双方
- 当前盘口或盘口占位
- “开始分析”主按钮
- 今日结论，例如“建议小额参与，避开超低赔率场”

分析结果区域：

- 今日稳胆 TOP3
- 今日价值投注 TOP3
- 今日避坑比赛
- 推荐串关

推荐串关分三档：

- 保守方案：2 串 1，示例投入 20 元
- 平衡方案：3 串 1，示例投入 20 元
- 冲高方案：4 串 1，示例投入 10 元

页面底部常驻风险说明：推荐仅用于辅助判断，不保证结果，不自动下注，示例投入不代表强制或正式投注建议。

## 6. 比赛详情

用户可以从比赛列表、榜单或串关方案进入比赛详情。

详情内容包括：

- 双方球队
- FIFA 排名
- 最近 5 场战绩
- 近期进球数
- 近期失球数
- 当前盘口或赔率
- 综合分
- 各评分维度分数
- 推荐方向
- 是否进入稳胆榜、价值榜或避坑榜
- AI 风格解释：为什么推荐、为什么不推荐、风险在哪里

详情页文案面向足球小白，避免堆砌专业术语。专业概念必须翻译成直接可理解的解释，例如“赔率太低，赢了也赚得少，所以不值得冒险”。

## 7. 推荐逻辑

核心计算由规则评分完成，AI 不参与核心计算。

每场比赛计算 0-100 的综合分，评分维度如下：

- 球队实力：30%
- 近期状态：25%
- 进攻能力：15%
- 防守能力：15%
- 历史交锋：5%
- 市场赔率：10%

### 7.1 稳胆榜

稳胆榜从综合分较高、风险较低、赔率不过分离谱的选项中生成。

排序因素：

- 综合分
- 风险等级
- 数据完整度
- 盘口或赔率是否异常

### 7.2 价值榜

价值榜关注“模型概率 vs 市场赔率隐含概率”的差异。

当模型认为某方向的胜出概率高于赔率隐含概率，并且风险不高于可接受阈值时，该选项可以进入价值榜。

### 7.3 避坑比赛

以下情况可进入避坑比赛：

- 实力差距过大但赔率过低
- 收益不足
- 数据缺失较多
- 盘口变化异常
- 模型分歧明显
- 风险等级高于推荐阈值

### 7.4 串关方案

串关方案从稳胆榜和价值榜中组合生成。

- 保守方案优先选择 2 个风险较低的选项
- 平衡方案选择 3 个风险可控且分散的选项
- 冲高方案选择 4 个包含一定价值机会的选项

串关方案必须显示风险等级，并明确金额是“示例投入”。

## 8. AI 职责

AI 只负责解释，不负责核心计算。

AI 风格文案职责：

- 解释为什么推荐
- 解释为什么不推荐
- 解释风险在哪里
- 将评分结果翻译成小白能理解的话
- 提醒用户控制投入和接受不确定性

第一版可用本地模板模拟 AI 文案，例如根据评分、风险和榜单类型生成解释句。后续可以替换为真实大模型生成，但输入必须来自已计算好的结构化结果。

## 9. 数据设计

第一版使用示例数据，字段按真实 API 接入预留。

核心数据对象：

- Match：比赛
- Team：球队
- TeamForm：近期状态
- Odds：赔率与盘口
- ScoreBreakdown：评分拆解
- Recommendation：推荐项
- ParlayPlan：串关方案

示例数据至少覆盖：

- 6 场今日比赛
- 每支球队 FIFA 排名
- 每支球队最近 5 场战绩
- 每支球队近期进球数和失球数
- 若干盘口与胜平负赔率
- 至少 1 场避坑比赛

## 10. 技术架构

建议使用 Vite + React + TypeScript 实现本地网页 App。

模块划分：

- `fixtures`：今日比赛、球队排名、近 5 场、赔率和盘口示例数据
- `services`：数据读取接口，第一版读取示例数据，后续替换真实 API
- `scoring`：核心评分计算
- `recommendations`：榜单和串关生成
- `explanations`：解释文案生成
- `ui`：首页、榜单、详情、空状态、风险提示

数据流：

```text
示例赛事数据
→ 评分模块
→ 推荐模块
→ 解释文案
→ 首页与详情展示
```

## 11. 错误与空状态

需要覆盖以下状态：

- 今日有比赛但尚未分析
- 今日有比赛且已生成分析
- 今日暂无世界杯赛事
- 部分盘口缺失
- 部分球队数据缺失
- 推荐结果不足 3 个

当数据缺失时，页面必须明确显示“部分数据缺失，建议谨慎参考”，不能假装数据完整。

## 12. 验收标准

第一版完成时应满足：

- 打开网页能看到今日比赛和“开始分析”
- 点击“开始分析”后出现稳胆 TOP3、价值 TOP3、避坑比赛和 3 个串关方案
- 每个推荐可以进入详情
- 详情中能看到球队数据、评分拆解、推荐理由和风险提示
- 页面明确显示“辅助决策，不保证结果，不自动下注”
- 串关金额显示为“示例投入”
- 数据接口边界清楚，后续可以替换成 API-Football 或 Football Data API
- 桌面和手机宽度下都能正常阅读，不重叠、不裁切
- 核心评分模块可以用测试验证排序和推荐结果

## 13. 后续扩展

第一版完成后，后续可按优先级扩展：

- 接入真实免费足球数据 API
- 增加真实盘口来源
- 增加历史比赛结果回测
- 增加用户自定义风险偏好
- 增加预算输入与资金分配
- 增加 AI 解释生成接口
- 增加比赛日历和历史推荐记录

这些扩展不进入 MVP 第一版。
